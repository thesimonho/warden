# Service Layer

Business logic layer — orchestrates engine, database, event store, and event logger without HTTP concerns. Both the HTTP server and direct Go library consumers call the same service methods. API contract types are defined in `api/` and re-exported via type aliases in `api_aliases.go`.

## Core

| File                   | Purpose                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `service.go`           | `Service` struct with lifecycle deps: `engine.Client`, `*db.Store`, `*eventbus.Store`, `*db.AuditWriter`, `*agent.Registry`, `*hook.Watcher`, `EventHandler` callback, `HomeDir`, `dockerDesktop bool` (set from `ServiceDeps.IsDockerDesktop`, used to trigger socket mount rewriting for Docker Desktop). Constructor `New(ServiceDeps)` wires everything. Manages session watcher lifecycle via `sessionWatchers map[string]*agent.SessionWatcher` and synchronous `sync.RWMutex`. Public method `GetProject(projectID) (*db.ProjectRow, error)` for lookups (returns `ErrNotFound` if not found). Session watcher methods called by `CreateContainer`, `DeleteContainer`, and on server startup (`ResumeSessionWatchers`).                                               |
| `api_aliases.go`       | Type aliases re-exporting `api.*` types as `service.*` for backward compatibility                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `errors.go`            | Service-layer error sentinels (`ErrNotFound`, `ErrInvalidInput`)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| `session_lifecycle.go` | `StartSessionWatcher(projectID, agentType, containerName, workspaceDir)` creates a session watcher and registers an event callback that bridges parsed JSONL events to container events (threading through `SourceLine`/`SourceIndex` for dedup). `StopSessionWatcher(projectID, agentType)` removes the watcher and records a cooldown timestamp. `RestartSessionWatcher()` stops and restarts. `ResumeSessionWatchers(ctx)` called on startup to recover watchers for running containers. `HandleContainerAlive(projectID, agentType, containerName)` reactive watcher start on first container event (covers restart/external creation edge cases). |
| `close.go`             | `Close()` graceful shutdown: stops all session watchers, cleans up resources (called by `warden.Close()`).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| `resolve.go`           | `resolveProject(projectID)` internal helper called by worktree/container methods to fetch `*ProjectRow` before delegating to internal methods. Returns `ErrNotFound` if project doesn't exist.                                                                                                                                                                                                                                                                                                                                                                                                                                                         |

## Domain Files

| File                | Purpose                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |

| `projects.go`       | `ListProjects(ctx) ([]engine.Project, error)`, `RemoveProject(projectID, agentType) (*ProjectResult, error)` (emits `project_removed` audit event; cleans up costs/audit when audit logging is off), `ResetProjectCosts(projectID, agentType) error` (emits `cost_reset` audit event), `PurgeProjectAudit(projectID, agentType) (int64, error)` (emits `audit_purged` audit event), `GetProject(projectID, agentType) (*db.ProjectRow, error)`, `StopProject(ctx, projectID, agentType) (*ProjectResult, error)` (takes project ID and agent type, resolves internally), `RestartProject(ctx, projectID, agentType) (*ProjectResult, error)` (takes project ID and agent type, resolves internally), `HandleContainerStale(projectID, agentType, containerName)` (writes audit entry with resolved project context — called by event bus stale callback; does NOT stop session watchers to prevent races during rapid container recreation), `HandleContainerAlive(projectID, agentType, containerName)` (session watcher startup callback). Cost overlay (DB → event store → agent config fallback) and DB metadata overlay applied at `ListProjects` time.                                                                                                                                                                                                                                                                                                                     |
| `budget.go`         | `PersistSessionCost(projectID, agentType, sessionID string, cost float64, isEstimated bool)` — single gateway for all cost writes (persists to DB keyed by `(projectID, agentType)`, triggers budget enforcement). `enforceBudget(projectID, agentType)` (unexported, called exclusively by `PersistSessionCost` — single `GetProject` call derives budget, container name, and container ID; executes configured actions: warn, stop worktrees, stop container). `GetEffectiveBudget(projectID, agentType) float64` (per-project-agent or global default). `effectiveBudgetFromRow(row)` (internal, computes budget from an already-fetched row). `GetDefaultProjectBudget() float64`. `IsOverBudget(projectID, agentType) bool` (checks cost + preventStart action for blocking restarts). `getBudgetActions()` reads and parses budget action settings.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| `containers.go`     | `CreateContainer(ctx, request) (*ContainerResult, error)` (creates project + container; calls `mergeRuntimeDomains` to append runtime-specific and system network domains to the allowed domains list for restricted mode), `DeleteContainer(ctx, projectID, agentType) (*ContainerResult, error)` (emits `container_deleted` audit event; `CreateContainer` emits `container_created` audit event, takes project ID + agent type and resolves internally), `InspectContainer(ctx, projectID, agentType) (*ContainerConfig, error)` (returns enabled access items as slice via `splitCSV`, takes project ID + agent type), `UpdateContainer(ctx, projectID, agentType, request) (*ContainerResult, error)` (takes project ID + agent type; `needsRecreation()` determines heavy vs light path — AllowedDomains changes are hot-reloaded via `ReloadAllowedDomains` in the light path), `ValidateContainer(ctx, name, image) (*ValidateContainerResult, error)`. Internal methods use `*db.ProjectRow` resolved via `resolveProject()`. Helper `projectRowFromRequest(req)` converts `CreateContainerRequest` to `ProjectRow`. `mergeRuntimeDomains(request)` calls `buildEffectiveDomains` to merge user-specified domains with system domains (for agent CLI downloads) and runtime-contributed domains (for package registries). `buildEffectiveDomains(userDomains, enabledRuntimes)` merges all three sources and deduplicates via `mergeDomainsDedup`. Database stores user-entered domains only; merged domains are only passed to Docker engine. `reconstructRequestFromRow(project, update)` rebuilds a full `CreateContainerRequest` from a `ProjectRow` for template write-back during updates. `HandleContainerStart(containerID, containerName)` — Docker events callback that checks the `recentlyCreated` sync.Map; skips network re-application for containers just created (already applied before start), and re-applies iptables for external restarts by merging current domains and calling `ApplyNetworkIsolation`. |
| `template.go`       | `.warden.json` project template read/write system. **Normalize helpers**: `normalizeAgentType`, `normalizeRuntimes`, `normalizeNetworkMode` — single source of defaults for agent type, runtimes, and network mode, used across template writes, recreation checks, DB persistence, and audit diffs. **Read**: `readProjectTemplate(dir)`, `parseTemplate(filePath)`, `sanitizeTemplate(tmpl)` (strips domains when not restricted, filters invalid runtime IDs), `ReadProjectTemplate(filePath)` (exported, for import feature). **Write**: `templateData` struct (narrow, normalized input — only template-relevant fields, excludes envVars/accessItems for security), `newTemplateData(req)` (normalizes `CreateContainerRequest`), `writeProjectTemplate(td)` (atomic write preserving other agent overrides). |
| `worktrees.go`      | `ListWorktrees(ctx, projectID, agentType) ([]engine.Worktree, error)`, `CreateWorktree(ctx, projectID, agentType, name) (*WorktreeResult, error)`, `ConnectTerminal(ctx, projectID, agentType, worktreeID) (*WorktreeResult, error)`, `DisconnectTerminal(ctx, projectID, agentType, worktreeID) (*WorktreeResult, error)`, `KillWorktreeProcess(ctx, projectID, agentType, worktreeID) (*WorktreeResult, error)`, `RemoveWorktree(ctx, projectID, agentType, worktreeID) (*WorktreeResult, error)`, `CleanupWorktrees(ctx, projectID, agentType) (*WorktreeResult, error)`, `GetWorktreeDiff(ctx, projectID, agentType, worktreeID) (*DiffResponse, error)`, `NotifyTerminalDisconnected(projectID, agentType, worktreeID)`. All methods accept project ID + agent type (strings) instead of `*db.ProjectRow`; resolve internally via `resolveProject()`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                       |
| `session_bridge.go` | `SessionEventToContainerEvent(agent.ParsedEvent, SessionContext) *event.ContainerEvent` — converts parsed JSONL events to eventbus events for SSE/audit pipeline. Threads `SourceLine` and `SourceIndex` through to the container event for audit log dedup hashing. All `ParsedEventType` constants are bridged (no unmapped types). Attaches event-specific data payloads: `ToolUseData`, `CostData`, `ToolUseFailureData`, `StopFailureData`, `PermissionRequestData`, `ElicitationData`, `TurnDurationData`, `SubagentData`, `ApiMetricsData`, `PermissionGrantData`, `ContextCompactData`, `SystemInfoData`. Helpers: `mapEventType` (ParsedEventType → ContainerEventType), `parseTimestamp`, `marshalData`.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| `settings.go`       | `GetSettings` (includes pinned CLI versions `ClaudeCodeVersion`/`CodexVersion` from `agent/versions.go`), `UpdateSettings` (runtime, auditLogMode, disconnectKey, defaultProjectBudget, budgetAction{Warn,StopWorktrees,StopContainer,PreventStart}), `GetDefaultProjectBudget`, `getBudgetActions`, `parseFloat`/`formatFloat` helpers for DB serialization                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| `audit.go`          | `GetAuditLog` (filtered audit events by category/projectID/worktree/source/level/time), `GetAuditSummary` (aggregate stats: sessions, tools, prompts, cost, top tools), `PostAuditEvent` (add custom audit entry), `DeleteAuditEvents` (scoped deletion with query filters), `GetAuditProjects` (distinct project names), `WriteAuditCSV` (CSV export for compliance). `auditEventsByCategory` maps 7 categories to event types — session (turn_complete, turn_duration, context_compact, system_info, terminal lifecycle, worktree lifecycle, stop_failure), agent (tool_use, tool_use_failure, permission_request/grant, subagent, elicitation, task_completed), prompt, config, budget, system (api_metrics, process_killed, container_created, container_deleted, container_updated, runtime_installed), debug (unmapped slog events). `StandardAuditEvents()` returns the standard-mode allowlist.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| `host.go`           | `GetDefaults(projectPath)` (returns `DefaultsResponse` with user `Mounts`, `EnvVars`, `RestrictedDomains` per agent type, `Runtimes` auto-detected from the project path, and `Template` from `.warden.json` if present), `ListDirectories(path, includeFiles)` (dirs-first sort; file mode for bind mount browser), `RevealInFileManager`, `ListRuntimes`. `userMounts` slice includes both `.claude` and `.codex` config directories (marked `required: true` — mandatory for auth, agent config, and session parsing; UI prevents removal). `RestrictedDomains` is the single source of truth for agent-scoped network allow lists — both webapp and TUI read from this API. |
