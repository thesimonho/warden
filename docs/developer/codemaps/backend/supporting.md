# Supporting Packages

Smaller backend packages that don't warrant individual files.

## Entry Points

- `warden.go` — top-level `Warden` type: `New(Options)` wires SQLite database, runtime detection, engine client, event bus pipeline, and service layer into a ready-to-use handle; calls `docker.Detect()` once at startup and passes `DockerInfo` and `BridgeIP` through to the service layer via `ServiceDeps`; calls `SetIsDesktop` on the engine client, `SetupBridgeFirewall` to create the host iptables chain for socket bridge ports, and `ResumeSocketBridges` to recover TCP bridge proxies for running containers. `Close()` shuts down all subsystems (idempotent via `sync.Once`). Fields: `Service` (primary interface for all operations), `Broker` (SSE event bus), `Engine` (container runtime client, advanced use), `DB` (SQLite store, advanced use), `Watcher` (file-based event watcher, advanced use), `DockerDesktop bool` (exposed so frontends can display runtime-specific guidance). All container/worktree operations are accessed through `w.Service`. Session watcher lifecycle is managed by the service layer: `service.StartSessionWatcher()` and `service.StopSessionWatcher()` handle JSONL session file monitoring. Watchers are stopped only by explicit lifecycle events (delete, stop, rename, recreate, shutdown) — not on heartbeat stale. This is the primary entry point for Go library consumers.

## internal/terminal/

WebSocket-to-PTY proxy for terminal connections.

| File                | Purpose                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| ------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `proxy.go`          | `Proxy` type with `ServeWS(mode)` — upgrades HTTP to WebSocket, captures tmux scrollback via `captureScrollback()` (runs `tmux capture-pane -p -S -5000`), sends it as a binary frame, then creates docker exec with TTY attached to tmux and bridges bytes bidirectionally. Scrollback replay fills the gap between the user's last disconnect and now. Handles resize via text frames, ping/pong heartbeat (30s), graceful close. PTY output is filtered through `AltScreenFilter` to enable scrollback. `mode` enum: `modeAgent` (default agent session `warden-<wid>`) or `modeShell` (auxiliary bash session `warden-shell-<wid>`). Includes `ServeShellWS()` helper and `bootstrapShell()` for lazy shell initialization. HTTP handlers (`handleTerminalWS`, `handleShellTerminalWS`) live in `internal/server/routes_streaming.go`. |
| `altscreen.go`      | `AltScreenFilter` — an `io.Reader` wrapper that strips alternate screen escape sequences (DECSET/DECRST 47, 1047, 1049) from the PTY output stream. Forces applications like Claude Code (via Ink) to render in the normal buffer where xterm.js scrollback works. Handles combined DECSET params and sequences split across read boundaries.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
