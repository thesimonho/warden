## Socket Bridge & Firewall

| File             | Purpose                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ---------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `firewall.go`    | Host iptables management for socket bridge TCP listeners via a dedicated `WARDEN-BRIDGE` chain. Uses short-lived Docker containers with host networking and `NET_ADMIN` to modify the host's iptables (not the container's). `SetupBridgeFirewall` creates and flushes the chain on startup (idempotent). `AddBridgeFirewallRule`/`AddBridgeFirewallRules` add ACCEPT rules for bridge TCP ports (batched to avoid per-port container overhead). `RemoveBridgeFirewallRule` removes a single port rule on bridge stop. `TeardownBridgeFirewall` flushes and removes the chain on graceful shutdown. All methods no-op on Docker Desktop (VM NAT handles forwarding). |
| `bridge_exec.go` | Docker exec management for socat bridges inside containers. `ExecSocatBridge(ctx, containerID, containerPath, port)` starts a detached socat process that creates a Unix socket at `containerPath` and forwards connections to `host.docker.internal:port` (the host-side TCP bridge listener). `KillSocatBridges(ctx, containerID)` kills all socat bridge processes inside the container (called before re-exec'ing bridges with new ports after server restart).                                                                                                                                                                                                  |

## Go Constants

- `terminalsDirSuffix = "/.warden/terminals"` — appended to workspace dir for terminal tracking
- `wardenWorktreesPrefixSuffix = "/.warden/worktrees/"` — appended to workspace dir for non-Claude agent worktrees
- `createTerminalScript = "/usr/local/bin/create-terminal.sh"` — in-container script to initialize tmux session
- `disconnectTerminalScript = "/usr/local/bin/disconnect-terminal.sh"` — in-container script to kill tmux session
- Method: `worktreesPrefixSuffix(ctx, containerID)` — returns agent-dependent worktree path prefix (`.claude/worktrees/` for Claude or `.warden/worktrees/` for other agents)

For container environment variables (`WARDEN_*`), see [container/environment.md](../container/environment.md).
