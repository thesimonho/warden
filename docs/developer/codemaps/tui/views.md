# TUI Views

All views implement the `View` interface.

## View Summary

| View                | Purpose                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `ProjectsView`      | Full-width table (`bubbles/table`) with columns: status dot, project name, runtime, worktree count, agent (8 chars, short label), cost. Row selection highlight. Actions: open (detail), edit (or create container for no-container projects), toggle (start/stop container), new, manage, refresh.                                                                                                                                      |
| `ManageProjectView` | Inline overlay dialog for project management with four independent checkboxes: Remove from Warden, Delete container, Reset cost history, Purge audit history. Purge requires typing "purge" to confirm. Ordered execution with concurrent data cleanup.                                                                                                                                                                                  |
| `ProjectDetailView` | Worktree list using `bubbles/list` with custom `worktreeDelegate` for 3-line items (title, branch, colored status). Actions: connect (agent terminal passthrough), shell (auxiliary bash terminal), disconnect, kill, remove, new (dialog), cleanup.                                                                                                                                                                                                                            |
| `ContainerFormView` | Create or edit container. Full field set matching the webapp: agent type (button-group selector, first field), name, path, skip permissions (agent-aware label), network mode, allowed domains, and collapsible Advanced section with access item selection, image, bind mounts, and environment variables. Read-only in edit mode. Uses `agent.AllTypes`, `agent.DisplayLabels`, and references `agent.Codex` for agent-specific logic. |
| `SettingsView`      | Settings menu (cursor navigation j/k, enter to activate): desktop notifications toggle (`notificationsEnabled`), event log toggle, runtime selector, disconnect key capture, default budget input, budget enforcement action toggles.                                                                                                                                                                                                    |
| `EventLogView`      | Event log entries filtered by level and source. Reverse chronological order. Actions: level filter, source filter, refresh, clear. Visible only when auditLogMode is Standard or Detailed.                                                                                                                                                                                                                                               |
| `AuditLogView`      | Audit log viewer with full filtering: category, level, source, project, time range. Auto-refresh (10s/30s/1m/5m). Scoped delete with type-to-confirm. Summary shows sessions, tools, prompts, cost, projects, worktrees.                                                                                                                                                                                                                 |

## View Files

| File                            | Purpose                                                                                                                                                                                                                                       |
| ------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `view_projects.go`              | `ProjectsView` — project list via `bubbles/table`. Displays status dot, project name, runtime, worktree count, agent (8-char short label via `agent.ShortLabel()`), cost. Actions: open, edit, toggle, manage, new, refresh.                  |
| `view_manage_project.go`        | `ManageProjectView` — inline overlay with four toggle checkboxes, type-to-confirm for audit purge, ordered execution, and OperationResultMsg handling.                                                                                        |
| `view_project_detail.go`        | `ProjectDetailView` — worktree list using `bubbles/list` with custom `worktreeDelegate`. Agent terminal passthrough (enter) or shell terminal (s) yields to `TerminalExecCmd`, returns on disconnect key. Helper `attachTerminal` and `attachShellTerminal` methods wire to Client interface.                                                                         |
| `view_container_form.go`        | `ContainerFormView` — state, Update, key handling. Includes agent type field (first in form). Mount/env inline editing with enter=save, esc=cancel. Calls `CheckContainerName` via Client interface to check name availability before submit; prompts user to confirm replacement if a Warden-managed container exists (via `ForceReplace` flag). Passes `AgentType` in submit payload. |
| `view_container_form_render.go` | Render logic: `buildFieldLines()` with cursor tracking for scroll-to-cursor, `fieldView()`, `appendListSection()`, helper functions. Agent type field rendered as button-group style toggle using `agent.AllTypes` and `agent.DisplayLabels`. |
| `view_container_form_help.go`   | Help keymaps for form modal states: `FormKeyMap`, `browsingHelpKeyMap`, `editingHelpKeyMap`, `inlineEditHelpKeyMap`, `formSelectionKeyMap`, `formWithRemoveKeyMap`.                                                                           |
| `view_settings.go`              | `SettingsView` — settings menu with cursor navigation and action items. Includes `settingsItemNotificationsEnabled` toggle for desktop notifications.                                                                                         |
| `view_eventlog.go`              | `EventLogView` — event log viewer (tab [3]). Fetches audit entries with optional filters.                                                                                                                                                     |
| `view_auditlog.go`              | `AuditLogView` — audit log viewer (tab [4]). Full filter set, auto-refresh via `tea.Tick`.                                                                                                                                                    |
